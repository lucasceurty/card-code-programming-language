01 — What is CardCode?

CardCode is a stack-based programming language where the source code is a physical deck of playing cards. Point a webcam at a row of cards, press Capture Line, and the system reads each card left-to-right and converts it into a token. Press End & Run when your program is complete — CardCode transpiles the token sequence to Python and executes it on the spot.

Programs are saved as .card files — plain text sequences of card shorthand that can be re-run any time from the command line.

02 — Try it yourself

The simplest program to test: build the string "Fizz" from ASCII values and print it. Lay these cards out one row at a time:

fizz.card
# Build 'F' (70 = 10 × 7), convert to char
10H 7H 3S JH

# Build 'i' (105 = 10×10 + 5), convert to char
10H 10H 3S 5H AS JH

# Build 'z' (122 = (10+10)×6 + 2), convert to char
10H 10H AS 6H 3S 2H AS JH

# Duplicate 'z', concat all four chars into "Fizz", store
8S JS JS JS 5C

# Load and print
7C JC

Run it from the UI — or from the command line:

python run.py programs/fizz.card

Expected output:

Fizz

git clone https://github.com/YOUR_USERNAME/cardcode

03 — The Card Map

Each card maps to exactly one instruction. The suit determines the category; the rank determines the specific command.

♥ Hearts — Integers
AH→PUSH_1
2H→PUSH_2
3H→PUSH_3
4H→PUSH_4
5H→PUSH_5
6H→PUSH_6
7H→PUSH_7
8H→PUSH_8
9H→PUSH_9
10H→PUSH_10
QH→PUSH_0
♠ Spades — Arithmetic
AS→ADD
2S→SUB
3S→MUL
4S→MOD
5S→EQ
6S→AND
7S→NOT
8S→DUP
9S→SWAP
10S→POP
♦ Diamonds — Control Flow
AD→FOR
2D→LOAD_CTR
3D→IF
4D→ELSE
5D→END
♣ Clubs — Registers
AC→STORE_A
2C→STORE_B
3C→LOAD_A
4C→LOAD_B
5C→STORE_STR_A
6C→STORE_STR_B
7C→LOAD_STR_A
8C→LOAD_STR_B

Face cards — String operations

JH  →  CHAR        (pop integer, push ASCII character)
JS  →  CONCAT      (pop two strings, push joined string)
JD  →  NUM_TO_STR  (pop integer, push as string)
JC  →  PRINT_STR   (pop string, print it)
QC  →  PRINT_NUM   (pop integer, print it)

04 — How Numbers Work

There are no numeric literals beyond 1–10. Every larger number is constructed by pushing heart cards and combining them with arithmetic. This makes number-building feel like actual card play. 

# The number 100
10H 10H 3S          →  push 10, push 10, MUL  =  100

# The number 70  (ASCII 'F')
10H 7H 3S           →  10 × 7  =  70

# The number 122  (ASCII 'z')
10H 10H AS 6H 3S 2H AS   →  (10+10) × 6 + 2  =  122

String building works the same way — construct the ASCII value, call JH (CHAR) to convert it to a character, then concatenate with JS.

05 — The .card File

Programs are saved as .card files. Each line of the file corresponds to one row of physical cards, written in shorthand notation. Comments start with #. The generated Python and output are appended as comments so the file is both human-readable and fully re-runnable.

Example: programs/fizz.card
# CardCode Program
# Generated: 2025-05-05T14:32:07
# Lines: 5  Tokens: 25

10H 7H 3S JH
10H 10H 3S 5H AS JH
10H 10H AS 6H 3S 2H AS JH
8S JS JS JS 5C
7C JC

# ── Generated Python ─────────────────────────────────
# _str_a = (chr((10 * 7)) + (chr(((10 * 10) + 5)) + ...))
# print(_str_a)

# ── Output ───────────────────────────────────────────
# Fizz

Re-run any saved program from the command line:

python run.py programs/fizz.card

06 — FizzBuzz

The canonical CardCode challenge. Lay out 25 rows of cards — the first eight build "Fizz" and "Buzz" from ASCII, the remainder implement the loop and conditional logic.

The full FizzBuzz program laid out as physical cards.
# Build "Fizz" → stored in Str register A
10H 7H 3S JH
10H 10H 3S 5H AS JH
10H 10H AS 6H 3S 2H AS JH
8S JS JS JS 5C

# Build "Buzz" → stored in Str register B
10H 6H 3S 6H AS JH
10H 10H 3S 10H AS 7H AS JH
10H 10H AS 6H 3S 2H AS JH
8S JS JS JS 6C

# Loop 1..100
10H 10H 3S AD

# FizzBuzz: (ctr%3==0) AND (ctr%5==0)
2D 3H 4S QH 5S 2D 5H 4S QH 5S 6S 3D
7C 8C JS JC
4D
# Fizz: ctr%3==0
2D 3H 4S QH 5S 3D
7C JC
4D
# Buzz: ctr%5==0
2D 5H 4S QH 5S 3D
8C JC
4D
# Else: print number
2D JD JC
5D 5D 5D 5D
